# Help Desk Ticket Classification
**Naive Bayes from scratch | Universidad Rafael Landivar - AI 2026**

## Description
This project implements a manual **Multinomial Naive Bayes** classifier for help desk tickets. It classifies support requests into the dataset's five categories:

- `Technical issue`
- `Billing inquiry`
- `Product inquiry`
- `Refund request`
- `Cancellation request`

The system includes:

- text preprocessing,
- Bag of Words representation,
- Laplace smoothing,
- log-probability inference,
- manual K-Folds cross validation,
- model persistence,
- and a Flask web interface for ticket submission.

## Dataset
The project uses the recommended dataset already included in the repository:

- File: `customer_support_tickets.csv`
- Text columns:
  - `Ticket Subject`
  - `Ticket Description`
- Label column:
  - `Ticket Type`

## Training
```bash
python train.py
```

This command:

1. loads the dataset,
2. preprocesses ticket subject and description,
3. runs manual 5-fold cross validation,
4. prints metrics,
5. trains the final model on the full dataset,
6. saves the model to `model.pkl`.

## Run the Web App
```bash

```

The application will be available at:

```text
http://localhost:5000
```

## Web Interface
Open `http://localhost:5000` in your browser and submit:

- `Ticket ID`
- `Subject`
- `Description`

The interface shows:

- the predicted category,
- the normalized text sent to the model,
- tokens used,
- class probabilities.

## API Endpoints
| Method | Route       | Description |
|--------|-------------|-------------|
| GET    | `/`         | Ticket web interface |
| GET    | `/health`   | Returns model status and vocabulary size |
| POST   | `/classify` | Classifies a ticket |

## Example Request
```bash
curl -X POST http://localhost:5000/classify \
     -H "Content-Type: application/json" \
     -d "{\"subject\":\"Duplicate charge on my account\",\"description\":\"I was charged twice for the same subscription and need help.\"}"
```

## Technologies
- **Python 3.11+**
- **NLTK** for stopwords and stemming only
- **Pandas** for dataset loading
- **Flask** for the web/API layer
- **No scikit-learn or equivalent classifiers**

## Model Notes
The classifier uses:

- **Bag of Words**
- **Laplace smoothing**
- **Log probabilities**
- **Manual stratified K-Folds**
- **Bigram features**

Because the dataset is in English, the best predictions are obtained when tickets are written in English as well.
